# OpenList 上传插件

MoviePilot 插件 — 监听整理完成事件，通过 OpenList API 将媒体文件上传到网盘（123云盘等）。

## 功能

- 监听 `TransferComplete` 事件，自动上传整理后的媒体文件
- 通过 OpenList 流式上传 API（`/api/fs/put`）上传
- 路径映射：本地路径 → 远端路径（支持多条规则）
- 文件过滤：按后缀、关键词、大小、类型（视频/字幕/NFO/图片）
- 远端已存在自动跳过
- 后台单线程异步上传，不阻塞 MP
- 失败重试（指数退避）
- 配置页支持完整设置
- 插件详情页展示队列统计
- MP 通知

## 配置

### 基础设置
| 配置项 | 说明 |
|--------|------|
| OpenList 地址 | OpenList 服务地址，如 `http://192.168.1.100:5244` |
| Token | OpenList 管理后台获取的 token（优先） |
| 用户名/密码 | 备选认证方式，token 失效时自动重新登录 |

### 路径映射
JSON 数组格式，每条规则：
```json
{
  "name": "电影",
  "local": "/media/Movies",
  "remote": "/123云盘/影视/电影"
}
```
按顺序匹配，取最长前缀匹配。

### 过滤规则
控制哪些文件上传到网盘，支持按后缀、关键词、文件大小、文件类型过滤。

### 重试设置
失败后自动重试，支持指数退避。

## 工作原理

```
MP 整理完成
    ↓ TransferComplete 事件
插件捕获事件 → 提取文件列表
    ↓
路径映射（本地 → 远端）
    ↓
文件过滤 → 生成 UploadTask
    ↓
入队 → 后台 Worker 异步上传
    ↓
OpenList API (/api/fs/put) → 123云盘
    ↓
状态记录（内存）→ 失败重试
```

## 注意事项

- 首次使用建议先在 OpenList 管理后台确认上传接口正常工作
- 大文件上传时注意 `timeout` 配置，建议设置 120 秒以上
- 插件重启会丢失内存中的任务队列（后续版本会加持久化）
